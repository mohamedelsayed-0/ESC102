Praxis I CTMFs

Backpack strap project

Stakeholder and requirement framing

Used in Praxis I

Frame

Definition

Making success criteria explicit by naming affected users, constraints, and failure cases before choosing a concept.

Evidence

The project shifted from "loose straps are annoying" to a more complete frame around comfort, snagging, access, adjustability, and manufacturability, which later justified the simulation parameters.

Utility and fit

This CTMF was useful because it prevented a superficial fix. It also counters my habit of jumping into modeling too early by forcing the decision criteria to exist first.

Failure modes and reminder

If the stakeholder set is too narrow, the later metrics will only optimize for one user experience. Use this before modeling, and revisit it whenever the team starts arguing over outputs that were never defined.

Prototype branching

Used in Praxis I

Diverge

Definition

Keeping multiple concept directions distinct long enough that they can be compared on more than aesthetic preference.

Evidence

The backpack work retained elastic, inside-bag adjustment, and shoulder-side adjustment as separate directions instead of collapsing immediately into one blended idea.

Utility and fit

This was useful because it delayed premature convergence and made later modeling meaningful. It is not my default strength, so it works as a deliberate correction to my preference for early structure.

Failure modes and reminder

Branching fails when concepts are kept alive only performatively and never compared on real criteria. Use it to protect real option space, then close it decisively once evidence is available.

Simplified biomechanical simulation

Used in Praxis I

Represent

Definition

A reduced model that turns prototype parameters into visible consequences such as moment arm, curvature, pressure, and deviation.

Evidence

The MATLAB tool modeled the lumbar region as a beam and exposed how different prototype parameters influenced deflection-related metrics.

Utility and fit

It helped because it gave the team a common language for comparing concepts. It fits my style well, but precisely because of that it needs discipline: representation can feel more certain than the evidence justifies.

Failure modes and reminder

This CTMF fails when the simplification is mistaken for direct user evidence. Use it to bound a decision, not to claim final ergonomic truth.

CIV102 CTMFs

Bridge project

Failure-mode decomposition

Used in CIV102

Frame

Definition

Breaking a structural design into distinct modes such as flexural tension, compression, buckling, shear, and glue failure before optimizing geometry.

Evidence

The hand calculations and Python analysis both tracked separate factors of safety across multiple modes and positions along the bridge.

Utility and fit

This CTMF made the bridge more legible because it turned "make it stronger" into specific mechanisms. It fits my analytical tendencies well, which is useful, but it can also make me underestimate fabrication as its own failure source.

Failure modes and reminder

If the decomposition ignores assembly quality or tolerance, the frame still remains too narrow. Use it together with an equally explicit constructability picture.

Cross-section tradeoff iteration

Used in CIV102

Converge

Definition

Iterating geometry by comparing competing consequences such as moment of inertia, shear flow, buckling risk, glue demand, and diaphragm spacing rather than optimizing one quantity.

Evidence

The section files and bridge analysis code treat geometry as a tunable design object whose properties can be recalculated across iterations.

Utility and fit

This CTMF is useful because it slows down single-metric optimization. It supports my natural tendency toward structure while forcing several capacities to be seen at once.

Failure modes and reminder

Iteration becomes false rigor if each geometry update is judged only analytically and never against the realities of cutting and gluing. Pair each iteration with a buildability check.

Engineering drawings as representation

Used in CIV102

Represent

Definition

Using drawings to make the assumed geometry, dimensions, and assembly relationships visible enough for a team to build from the same picture.

Evidence

The final bridge drawings translate section assumptions and spacing into an artifact the team could fabricate from rather than infer.

Utility and fit

This CTMF helped because it turned hidden assumptions into shared geometry. It also corrects a weakness in my process by forcing communication, not just calculation.

Failure modes and reminder

Drawings can look complete even when they omit assembly difficulty or tolerance risk. Use them early enough that they still have the power to change the design.

Gaussian Project CTMFs

Independent analytic work

Invariant reduction under symmetry

Used in Gaussian project

Frame

Definition

Reframing the problem so a symmetric setting reduces the entanglement condition to a simpler, analytically invertible form.

Evidence

The project README and benchmark code both center the move from repeated spectrum evaluation to parameter-level threshold expressions.

Utility and fit

This CTMF is one of my strongest natural tools. It created the main research leverage in the project by turning a computational task into a design constraint. It also embodies a risk: I can narrow scope too quickly when a symmetry is available.

Failure modes and reminder

Reduction fails when its assumptions are left implicit or treated as universal. State what the symmetry buys, and state what it excludes.

Parameter sweeps across channel regimes

Used in Gaussian project

Diverge

Definition

Exploring ranges of transmissivity, thermal occupancy, gain, and squeezing instead of relying on a few selected parameter points.

Evidence

The benchmark script generates multiple scenario families and heatmaps, making variation itself part of the argument.

Utility and fit

This CTMF widens the space after a strong abstraction has already appeared. It is especially helpful for me because it resists premature closure by forcing the reduced representation to survive more than one hand-picked example.

Failure modes and reminder

Sweeps can create false confidence if the ranges are arbitrary or too convenient. Pick ranges that reflect the real question you are asking, not just what makes a clean figure.

Exact-versus-bound benchmarking

Used in Gaussian project

Converge

Definition

Testing whether a simpler bound is good enough by comparing it directly to an exact quantity before using it as a design shortcut.

Evidence

The pure-loss and thermal-loss plots compare exact log-negativity to the reference bound and show where the approximation has visible slack.

Utility and fit

This CTMF is what makes the abstraction accountable. It is useful because it forces my preferred simplifications to earn permission rather than taking it for granted.

Failure modes and reminder

Benchmarking fails if the comparison family is too narrow. Any shortcut that will guide later decisions needs to be tested against the regime where it will actually be used.
